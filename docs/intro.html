Introduction

Features

The Usual Nuts & Bolts

Spinna comes with many of the features you would expect from an independant framework. Support for reading HTTP requests and writing HTTP responses, sessions, templates, validation etc. There are two things however, that are absent: databases and authorisation.

There are no database features because no aspect of Spinna requires one. You use your own favorite database code. I have written a set of database classes and you are welcome to use them, but it is by no means obligatory.

Authorisation can be done in many different ways, and I have not yet come across a satisfactory universal solution. So I provide a means by which you can supply your own authorisation mechanism, and if I come across something that can be used, I will consider including it in a future release.

Server Independance

Spinna is desigend to not tie you down to any particular type of web server. By creating an appropriate adaptor, you can use Spinna with any web server "front end". Spinna comes with three adaptors. One to allow execution from the console, another to interface with FCGI, and one to interface with Vibe-d.

Compile Time Router

Spinna's makerouter utitlity is used to generate D code that is compiled into the project to perform the routing tasks. Thus the router is direct binary which would be faster than run time created objects.

Pagebuilder

Pagebuilder is a set of classes to help you to construct HTML pages. While not trying to hide HTML from you, it does seek to take much of the boilerplate burden away from you.

Pagebuilder makes use of the fact that HTML is heirarchical in nature. So by using classes that derive from the Component interface, you can construct a heirarchy of objects, and then have them generate the HTML to be sent to output.

Thus you can create classes as reusable components, to encapsulate boilerplate, or simply to breakdown a complex page into smaller chunks.

Pagebuilder supplies classes for common components such as paginated lists, forms and widgets.

Templates

The ability to embed code inside plain text is popular with website programming, and Spinna provides its own template engine to do this. Similar in functionality to EmbD and Temple, Spinna's template engine is more specialised to work with Pagebuilder.
